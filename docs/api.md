# 03 — API Specification

Base path: **`/api/v1`**. JSON in/out. Auth via `Authorization: Bearer <access>`.
Interactive docs auto-generated by FastAPI at `/docs` (Swagger) and `/redoc`.

## Conventions

- **Auth levels:** `public` · `user` (any authenticated) · `owner` (resource
  owner or admin) · `admin`.
- **Pagination:** `?page=1&page_size=20` (cap 100). List responses:
  `{ "items": [...], "total": N, "page": 1, "page_size": 20 }`.
- **Errors:** `{ "error": { "code": "ITEM_NOT_FOUND", "message": "...", "details": {} } }`
  with appropriate HTTP status (400/401/403/404/409/422/429/500).
- **IDs:** UUID strings. **Timestamps:** ISO-8601 UTC.
- **Rate limits:** stricter on `auth/*` and write endpoints (per-IP + per-user).

## Auth — `/auth`

| Method | Path | Auth | Body / Params | Returns |
|--------|------|------|---------------|---------|
| POST | `/auth/register` | public | `email, password, full_name, phone?` | `201` user + tokens |
| POST | `/auth/login` | public | `email, password` | `access_token, refresh_token, user` |
| POST | `/auth/refresh` | public | `refresh_token` | new token pair (old refresh rotated/revoked) |
| POST | `/auth/logout` | user | `refresh_token` | `204`; revokes refresh token |
| GET | `/auth/me` | user | — | current user profile |
| PATCH | `/auth/me` | user | `full_name?, phone?, avatar_url?` | updated profile |
| POST | `/auth/verify-email` | public | `token` | `200` marks `is_verified` |
| POST | `/auth/forgot-password` | public | `email` | `202` (always, no user enumeration) |
| POST | `/auth/reset-password` | public | `token, new_password` | `200` |

Access token TTL ~15 min; refresh TTL ~30 days with rotation + reuse detection.

## Items (lost & found) — `/items`

| Method | Path | Auth | Notes |
|--------|------|------|-------|
| POST | `/items` | user | Create report. Body: `type(lost\|found), title, description, category_id?, color?, brand?, location_text?, location_lat?, location_lng?, event_date`. Returns `201` item (`status=open`, `processing_status=pending`); enqueues embedding+matching. Clients can poll `GET /items/{id}` for `processing_status` to reach `ready`. |
| GET | `/items` | public | Browse/search. Filters: `type, status, category_id, q (keyword), color, date_from, date_to, lat&lng&radius_km, user_id, sort`. Paginated. |
| GET | `/items/{id}` | public | Item detail with images + reporter (contact info gated to authenticated users). |
| PATCH | `/items/{id}` | owner | Update editable fields; re-enqueues embedding if text/category changed. |
| DELETE | `/items/{id}` | owner | Soft-close — `status=closed`, `closed_reason=withdrawn` (no hard delete). |
| POST | `/items/{id}/resolve` | owner | Mark recovered — `status=closed`, `closed_reason=recovered`. |
| POST | `/items/{id}/rematch` | owner | Force a fresh matching pass. |
| GET | `/items/{id}/matches` | owner | Ranked match suggestions for this item (see below). |

### Images — `/items/{id}/images`

| Method | Path | Auth | Notes |
|--------|------|------|-------|
| POST | `/items/{id}/images` | owner | `multipart/form-data` (1..N files). Validates MIME/magic-bytes/size, stores via `StorageBackend`, enqueues image embedding. Returns image records. |
| PATCH | `/items/{id}/images/{image_id}` | owner | Set `is_primary`. |
| DELETE | `/items/{id}/images/{image_id}` | owner | Remove image + storage object. |

`GET /media/{storage_key}` (dev) serves files with an ownership/visibility check;
in prod the API returns short-lived **signed S3 URLs** instead.

### `GET /items/{id}/matches` — response shape
```jsonc
{
  "item": { "id": "…", "type": "lost", "title": "Black leather wallet" },
  "matches": [
    {
      "match_id": "…",
      "candidate_item": { "id": "…", "type": "found", "title": "…",
                          "primary_image_url": "…", "location_text": "…",
                          "event_date": "2026-06-07" },
      "text_score": 0.83,
      "image_score": 0.91,
      "combined_score": 0.88,
      "confidence": 0.86,          // shown as 86%
      "status": "suggested",
      "explanation": ["same category", "image strongly similar",
                      "found 1 day after lost"]
    }
  ]
}
```
Ordered by `confidence` desc.

## Matches — `/matches`

| Method | Path | Auth | Notes |
|--------|------|------|-------|
| GET | `/matches/{id}` | owner | Single match detail (both items + scores). |
| POST | `/matches/{id}/confirm` | owner | Confirm. Match `status=confirmed`, both items → `claimed`, writes positive `match_feedback`, notifies counterpart. (Either owner then calls `/items/{id}/resolve` to close as recovered.) |
| POST | `/matches/{id}/reject` | owner | `status=rejected`, writes negative `match_feedback`. |
| POST | `/matches/{id}/feedback` | owner | `is_correct, comment?` explicit feedback for the learning loop. |

## Admin — `/admin`

All require `admin`; every mutating call writes an `admin_actions` audit row.

| Method | Path | Notes |
|--------|------|-------|
| GET | `/admin/stats` | KPIs: counts of users/items by type/status, matches, confirm rate, queue depth. |
| GET | `/admin/users` | List/search users; filters `role, is_active, q`. |
| PATCH | `/admin/users/{id}` | Change `role`, ban/unban (`is_active`), `verify`. |
| GET | `/admin/items` | All items incl. closed; moderation filters. |
| DELETE | `/admin/items/{id}` | Hard moderation removal (with `reason`). |
| GET | `/admin/matches` | Inspect matches; filter by confidence/status. |
| GET | `/admin/actions` | Read the audit log. |

## Endpoint → entity → service map

| Endpoint group | Primary entities | Service |
|----------------|------------------|---------|
| `/auth` | users, refresh_tokens | `auth_service` |
| `/items`, `/items/{id}/images` | items, item_images, categories | `item_service`, `image_service` |
| `/items/{id}/matches`, `/matches` | matches, match_feedback | `matching_service` |
| `/notifications` | notifications | `notification_service` |
| `/admin/*` | users, items, matches, admin_actions | `admin_service` |
